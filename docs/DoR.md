# Definition of Ready

Hieronder volgt de Definition of Ready. Dit is een checklist van criteria waaraan een taak of user story moet voldoen voordat het team ermee aan de slag kan. 
Het doel van de DoR is om ervoor te zorgen dat alle benodigde informatie en voorbereidingen aanwezig zijn, zodat het team zonder obstakels kan beginnen.

1. **Duidelijke Beschrijving**
    * De user story of taak bevat een duidelijke en volledige beschrijving van wat er moet worden gedaan.

2. **Acceptatiecriteria geformuleerd**
    * Er zijn acceptatiecriteria opgesteld die precies beschrijven wanneer de taak als "klaar" wordt beschouwd.

3. **Geen blokkerende issues**
    * Er zijn geen openstaande afhankelijkheden, blokkades of technische belemmeringen die de voortgang zouden kunnen hinderen.

4. **Architectuur**
    * Domein model moet opgesteld zijn voor code first database.
